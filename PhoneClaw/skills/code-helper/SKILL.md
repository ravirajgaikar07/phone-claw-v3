When helping with code:
- Always show the complete solution, not just fragments
- If editing an existing file, use read_file first to see current content
- After writing code, verify it by reading the file back
- For Python: prefer simple, readable code. Follow PEP 8
- For shell scripts: add error handling (set -e) and comments
- When debugging: check error messages carefully, suggest specific fixes
- If asked to create a project, set up proper structure (README, requirements, etc.)
